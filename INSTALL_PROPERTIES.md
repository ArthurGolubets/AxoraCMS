# Установка системы свойств товаров

## Шаг 1: Запуск миграций

Перейдите в корневую директорию проекта Laravel и выполните:

```bash
php artisan migrate
```

Это создаст две новые таблицы:
- `t_catalog_properties` - для хранения свойств категорий
- `t_product_property_values` - для хранения значений свойств товаров

## Шаг 2: Проверка установки

После запуска миграций проверьте, что таблицы созданы:

```bash
php artisan migrate:status
```

Вы должны увидеть в списке:
```
Ran  2026_03_29_000090_create_t_catalog_properties_table
Ran  2026_03_29_000091_create_t_product_property_values_table
```

## Шаг 3: Пересборка фронтенда (если необходимо)

Если вы используете сборку Vue компонентов:

```bash
npm run dev
# или для продакшена
npm run build
```

## Проверка работы

### В категориях:
1. Откройте любую категорию в админке
2. Перейдите на вкладку **"Свойства"**
3. Добавьте свойство (например: "Цвет", тип: "Строка")
4. Сохраните категорию

### В товарах:
1. Откройте товар, который принадлежит к категории со свойствами
2. Перейдите на вкладку **"Свойства"**
3. Вы должны увидеть все свойства категории (включая унаследованные)
4. Заполните значения свойств
5. Сохраните товар

### Проверка наследования:
1. Создайте родительскую категорию "Товары для дома" с свойством "Материал"
2. Создайте дочернюю категорию "Посуда" внутри "Товары для дома"
3. Добавьте в "Посуда" свойство "Объем"
4. Создайте товар в категории "Посуда"
5. В товаре должны быть доступны оба свойства: "Материал" (унаследовано) и "Объем"

## API для поиска по свойствам

Пример использования в коде:

```php
use HolartWeb\AxoraCMS\Services\CatalogService;

$service = new CatalogService();

// Поиск товаров по свойствам
$products = $service->searchProductsByProperties([
    'color' => 'red',    // по коду свойства
    'size' => 'XL'
], $catalogId = 5);

// Или по ID свойств
$products = $service->searchProductsByProperties([
    15 => 'red',   // где 15 - это property_id
    16 => 'XL'
]);
```

## Возможные проблемы

### Проблема: Вкладка "Свойства" не отображается
**Решение**:
- Проверьте, что миграции запущены
- Очистите кеш браузера
- Пересоберите фронтенд: `npm run dev`

### Проблема: Свойства не сохраняются
**Решение**:
- Проверьте, что в таблице `t_catalog_properties` есть записи
- Проверьте консоль браузера на наличие ошибок JavaScript
- Проверьте логи Laravel на наличие ошибок валидации

### Проблема: Не видны унаследованные свойства
**Решение**:
- Убедитесь, что категории правильно связаны через `parent_id`
- Проверьте, что свойства добавлены в родительскую категорию

## Структура базы данных

### Таблица t_catalog_properties
| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| catalog_id | bigint | ID категории |
| code | varchar | Код свойства (уникален в рамках категории) |
| name | varchar | Название свойства |
| type | enum | Тип: string, text, number |
| is_multiple | boolean | Множественное значение |
| sort_order | integer | Порядок сортировки |

### Таблица t_product_property_values
| Поле | Тип | Описание |
|------|-----|----------|
| id | bigint | Первичный ключ |
| product_id | bigint | ID товара |
| property_id | bigint | ID свойства |
| value | text | Значение (JSON для множественных) |
