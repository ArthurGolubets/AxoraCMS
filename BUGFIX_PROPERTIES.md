# Исправление проблем с зависанием при работе со свойствами

## Проблемы которые были исправлены:

### 1. Зависание при создании категории
**Проблема**: При создании новой категории страница зависала в бесконечной загрузке.

**Причина**:
- Компоненты со свойствами рендерились даже когда вкладка была скрыта (использовался `v-show`)
- При попытке сохранения пустых свойств возникали ошибки валидации

**Исправление**:
- ✅ Заменили `v-show` на `v-if` для ленивой загрузки компонентов
- ✅ Добавили проверку пустых свойств в контроллерах (пропускаются свойства с пустым code или name)

### 2. Зависание при нажатии "Добавить свойство"
**Проблема**: При нажатии кнопки "Добавить свойство" в категории или товаре страница зависала.

**Причина**: Бесконечный цикл обновлений между родительским и дочерним компонентами:
```
1. properties watcher эмитит update:properties
2. form.properties обновляется в родителе
3. initialProperties watcher видит изменение
4. properties обновляется снова
5. GOTO 1 (бесконечный цикл)
```

**Исправление**:
- ✅ Удалили `immediate: true` из watcher'а `initialProperties`
- ✅ Добавили флаг `isUpdatingFromParent` для предотвращения циклических обновлений
- ✅ Инициализация данных перенесена в хук `mounted()`
- ✅ Добавлена проверка на реальное изменение данных перед обновлением

## Тестирование

### Проверьте что работает:

1. **Создание категории**
   - Откройте "Создать категорию"
   - Заполните основные поля
   - Сохраните (не добавляя свойства)
   - ✅ Должно сохраниться без зависаний

2. **Добавление свойств в категорию**
   - Откройте категорию на редактирование
   - Перейдите на вкладку "Свойства"
   - Нажмите "Добавить свойство"
   - ✅ Должна появиться новая строка без зависания
   - Заполните код и название
   - Нажмите "Добавить свойство" еще раз
   - ✅ Должна добавиться еще одна строка
   - Сохраните категорию
   - ✅ Свойства должны сохраниться

3. **Редактирование свойств**
   - Откройте категорию со свойствами
   - Измените название свойства
   - ✅ Изменения должны сохраниться без зависания

4. **Свойства в товарах**
   - Создайте товар в категории со свойствами
   - Перейдите на вкладку "Свойства"
   - ✅ Должны отобразиться все доступные свойства
   - Заполните значения
   - Сохраните товар
   - ✅ Значения должны сохраниться

5. **Наследование свойств**
   - Создайте родительскую категорию со свойством "Материал"
   - Создайте дочернюю категорию
   - В дочерней категории на вкладке "Свойства" в блоке "Унаследованные свойства"
   - ✅ Должно отображаться свойство "Материал"

## Техническая информация

### Паттерн предотвращения циклов в Vue

```javascript
data() {
  return {
    localData: [],
    isUpdatingFromParent: false  // Флаг блокировки
  }
},
mounted() {
  // Инициализация при монтировании
  this.isUpdatingFromParent = true
  this.localData = JSON.parse(JSON.stringify(this.propData))
  this.$nextTick(() => {
    this.isUpdatingFromParent = false
  })
},
watch: {
  propData: {
    handler(newVal) {
      // Обновляем только если изменения пришли извне
      if (!this.isUpdatingFromParent) {
        this.isUpdatingFromParent = true
        this.localData = JSON.parse(JSON.stringify(newVal))
        this.$nextTick(() => {
          this.isUpdatingFromParent = false
        })
      }
    }
  },
  localData: {
    deep: true,
    handler(newVal) {
      // Не эмитим если обновляемся от родителя
      if (!this.isUpdatingFromParent) {
        this.$emit('update:data', newVal)
      }
    }
  }
}
```

Этот паттерн предотвращает циклы при двустороннем биндинге между родителем и дочерним компонентом.
